# Proposal Rubric

Use this as a quick screen for project quality.

| Category | Strong | Weak |
| --- | --- | --- |
| Problem | Real users and a meaningful need | Vague or trivial |
| Scope | Enough work for 6-7 students | Too small or unfocused |
| Complexity | Real engineering decisions and integration | Mostly simple CRUD |
| Collaboration | Multiple dependent workstreams | Easy to split into isolated tasks |
| Requirements | Clear constraints with room for design choices | Either unclear or over-prescribed |
| Feasibility | Realistic MVP in one semester | Unrealistic scope |

## Bottom line

A strong Capstone project should feel like a real engineering initiative, not a homework assignment.

<details>
<summary>What “strong” usually looks like</summary>

* Clear stakeholder involvement
* Real technical tradeoffs
* Milestones that can be demoed
* A realistic MVP
* Work that produces portfolio-quality outcomes

</details>
