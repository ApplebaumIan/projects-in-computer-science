# Stakeholder Guidance

Stakeholders should define the problem, constraints, and priorities.

Students should design and build the solution.

The professor should guide scope, risk, and engineering quality.

## Stakeholders should do

* Explain the real problem
* Identify users and goals
* Clarify constraints and priorities
* Give regular feedback
* Attend demos or milestone reviews

## Stakeholders should not do

* Micromanage implementation
* Dictate every technical choice
* Treat students like contract developers
* Continuously expand scope without prioritizing

<details>
<summary>Examples of acceptable constraints</summary>

* Must support Android and iOS
* Must integrate with existing hardware
* Must use a specific API or data source
* Must support offline operation
* Must meet accessibility requirements

</details>

<details>
<summary>What students are expected to own</summary>

* Architecture
* Database and API design
* Technology choices
* Testing strategy
* Deployment approach
* Day-to-day project management

</details>
