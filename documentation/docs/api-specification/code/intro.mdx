# Internal Code Contracts

## What are they?
Internal code contracts are a set of rules and expectations that govern how different components of a software system should interact with each other. They define the responsibilities, inputs, outputs, and behaviors of various modules, functions, or classes within the codebase. These contracts help ensure that different parts of the system can work together seamlessly and that any changes made to one component do not break the functionality of others.

An Example of this that you should be familar with is JavaDoc. JavaDoc is a documentation tool for Java that allows developers to create API documentation in a standardized format. It uses special comments in the code to generate HTML documentation that describes the classes, methods, and fields in a Java program. This documentation serves as an internal code contract by providing clear information about how to use the various components of the codebase, what parameters they expect, and what they return. By adhering to these contracts, developers can ensure that their code is maintainable, understandable, and less prone to errors when integrated with other parts of the system.

**For each class define the data fields, methods.**
- The purpose of the class.
- The purpose of each data field.
- The purpose of each method
- Pre-conditions if any.
- Post-conditions if any.
- Parameters and data types
- Return value and output variables
- Exceptions thrown\* (PLEASE see note below for details).
